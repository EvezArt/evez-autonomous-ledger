# Universal Intelligence Preservation

## Intent
Never collapse intelligence into a single provider, a single prompt, or a single volatile runtime trace.
The operator should preserve reasoning continuity, artifact continuity, and fallback continuity across every LLM surface it can access.

## Multipliers
Intelligence is multiplied by:
- provider diversity
- clear task classification
- preserved summaries
- preserved reason traces
- preserved artifacts
- governed fallback selection

## Live Policy
The runtime should:
1. classify the task
2. select a primary provider by capability fit and multiplier
3. define fallback providers
4. preserve summary, reason trace, and artifacts regardless of primary provider choice

## Universal Retention Contract
For every routed task preserve:
- summary
- reason trace
- produced artifacts
- proposal/approval state where relevant

## Governance
High-risk financial or trading execution is not an intelligence multiplier by itself. It remains approval-gated.
The multiplier is in better preparation, better comparison, and better preservation of context before action.

## Engineering Targets
- expose provider registry
- expose routing plan endpoint
- store retention artifacts durably when the runtime persistence layer is upgraded
- keep route selection explainable
